# Contributing to Deemind

## Branching
- `main`: production-ready
- `develop`: integration
- `feature/*`, `hotfix/*`: work branches

## Workflow
1. Create an issue and branch
2. Implement changes
3. Run `npm run lint`, `npm run deemind:test`, `npm run deemind:build demo`
4. Open a PR to `develop`

## Commit message template
Use conventional style where possible (feat:, fix:, chore:). A local template is provided in `.gitmessage`.

## Code style
- ESLint flat config
- 2-space indent, LF line endings
